## 主要な関数（main.gs）

### `doGet()`
GASウェブアプリのエントリーポイント。`index.html` を返す。

### `mainProcess(data)`
背景削除のメイン処理。

**フロー：**
1. タイムスタンプとpublic_id（`sizzle_${timestamp}`）を生成
2. SHA-1署名を生成（`generateSignature()`）
3. Cloudinary APIへ画像をアップロード（`background_removal: 'cloudinary_ai'`）
4. アップロード後、Admin APIでポーリング（3秒×最大10回）
5. `status === 'complete'` になったらPNG URLを返す

**ポーリングが必要な理由：**  
`cloudinary_ai` の背景削除は非同期処理のため、アップロード直後のURLは元画像のまま。Admin APIで完了を確認してから返す必要がある。

### `compositeProcess(data)`
背景合成処理。**アップロード不要**のURL変換方式。

**プリセット背景の場合（`bgType: 'preset'`）：**
```
https://res.cloudinary.com/{cloudName}/image/upload/
  b_rgb:{bgColor},c_pad,w_{w},h_{h},f_png/
  l_{fgPublicId},w_{w},h_{h},c_fit/fl_layer_apply,g_center/
  {fgPublicId}
```

**カスタム背景の場合（`bgType: 'custom'`）：**
```
https://res.cloudinary.com/{cloudName}/image/upload/
  w_{w},h_{h},c_fill/
  l_{fgPublicId},w_{w},h_{h},c_fit/fl_layer_apply,g_center/
  {bgPublicId}
```

取得した画像をGAS側でBase64化してフロントに返す（CORS回避）。

### `fetchImageAsBase64(imageUrl)`
ダウンロード用プロキシ。GASサーバー経由でCloudinaryから画像を取得し、Base64 DataURLで返す。フロントのCORS制限を回避するために必要。

### `uploadBackground(data)`
カスタム背景画像をCloudinaryにアップロード。`public_id: sizzle_bg_{timestamp}` で保存。

### `generateSignature(params, apiSecret)`
Cloudinary Signed モード用のSHA-1署名を生成。パラメータをアルファベット順に並べて `&` 連結し、末尾にAPIシークレットを付加してHash化。

## 解決した技術課題

### 1. `Display name cannot contain slashes` エラー
**原因：** Cloudinaryのアップロードプリセットが自動生成する Public ID にスラッシュ（日付フォルダ形式）が含まれていた。  
**解決：** `public_id: sizzle_${timestamp}` を明示指定することで自動生成を無効化。

### 2. 背景削除後も画像が変わらない
**原因：** `cloudinary_ai` は非同期処理。アップロード完了 ≠ 背景削除完了。  
**解決：** Admin APIのリソース取得エンドポイントで `status` をポーリング（3秒×10回）。

### 3. ダウンロードボタンがCORSエラー
**原因：** Cloudinaryドメインからブラウザへの直接Fetchがクロスオリジンでブロックされる。  
**解決：** `fetchImageAsBase64()` でGASサーバーを中継し、DataURL形式でフロントに返す。

### 4. 背景合成でアップロード不要にする
**原因：** 元の実装は `cloudinary://` URLを使って既存画像を参照していたが、GASの `UrlFetchApp` では無効なURL形式。  
**解決：** Cloudinaryの変換URL（Transformation URL）を直接構築する方式に変更。新規アップロード不要。

## デプロイ手順

1. GASエディタ（[script.google.com](https://script.google.com)）を開く
2. `main.gs` と `index.html` を各ファイルに貼り付けて保存
3. 「デプロイ」→「新しいデプロイ」→「ウェブアプリ」
4. アクセス権：「全員」 / 実行ユーザー：「自分」
5. 発行されたURLをブックマーク

**更新時：** 「デプロイ」→「デプロイを管理」→ 鉛筆アイコン → バージョン：「新バージョン」→「デプロイ」
